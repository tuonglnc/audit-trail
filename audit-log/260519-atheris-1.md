# 📌 1. Context — Atheris dùng để làm gì?

Python thường được xem là “memory-safe”, nhưng thực tế vẫn có nhiều vấn đề:

## 1.1. Logic Bugs

Ví dụ:

- `ValueError`
- `KeyError`
- `IndexError`

khi nhận input lạ từ người dùng.

---

## 1.2. Native Extensions

Nhiều thư viện Python thực chất gọi xuống code C/C++:

- `numpy`
- `cryptography`
- image parsers
- compression libs

Nếu phần native code có:

- Buffer Overflow
- Use-After-Free

=> toàn bộ process Python có thể crash.

---

## 1.3. Atheris giải quyết thế nào?

Atheris dùng:

```text
Coverage-guided Fuzzing
```

Tức là:

- sinh input
- theo dõi nhánh code mới
- mutate dữ liệu thông minh hơn
- cố đi sâu nhất vào chương trình

---

# 🧨 2. Threat Model — Nếu không có Atheris?

Nếu chỉ dùng:

- Unit Test
- SAST
- LLM review

thì sẽ khó cover hết các trường hợp input thực tế.

---

## 2.1. Các lỗi có thể xảy ra

### 🔥 RCE

Native extensions có thể dính:

- Buffer Overflow
- UAF
- Integer Overflow

---

### 💥 DoS

Unhandled exceptions làm service crash liên tục.

---

### 🕳️ Validation Bypass

Payload đặc biệt có thể bypass:

- regex
- validation logic
- sanitize rules

---

## 2.2. Giới hạn của LLM

LLM:

- đọc code tốt
- nhưng không execute thật
- không mutate input
- không tạo PoC thật

Atheris thì ngược lại:

```text
fuzz → crash → dump artifact → reproduce
```

---

## 2.3. LLM + Atheris

Flow rất mạnh:

```text
LLM đọc code
↓
Sinh fuzz harness
↓
Atheris fuzz thật
↓
Confirm bug bằng crash
```

# ⚙️ 4. Mechanism — Atheris hoạt động như thế nào?

Atheris là bridge giữa:

| Thành phần | Vai trò |
|---|---|
| LibFuzzer | mutate input |
| Python VM | chạy code |
| Atheris | gửi coverage về LibFuzzer |

---

## 4.1. Ý tưởng cốt lõi

LibFuzzer chỉ hiểu:

- machine code
- LLVM coverage

Trong khi Python chạy bằng:

```text
Bytecode + Python VM
```

=> LibFuzzer “mù” với Python.

Atheris giải quyết bằng:

```text
Bytecode Instrumentation
```

---

### 🔄 Flow tổng quát

```text
LibFuzzer
↓
Sinh raw bytes
↓
Atheris
↓
Đưa vào Python
↓
Python chạy code
↓
Coverage signal trả ngược về LibFuzzer
```

---

### 🔥 Ví dụ thực tế

```python
def login(data):
    if data.startswith(b"ADMIN"):
        if b"ROOT" in data:
            crash()
```

Ban đầu fuzzer random lung tung:

```text
hello
123123
AAAA
```

Sau nhiều lần mutate:

```text
ADMINxxxx
ADMIN_ROOT
```

=> cuối cùng trigger được crash.

---

## 4.2. Bytecode Instrumentation

Python source:

```python
x = a + b
```

được compile thành:

```text
LOAD_FAST
BINARY_ADD
STORE_FAST
```

---

### Atheris làm gì?

Khi gọi:

```python
atheris.instrument_all()
```

Atheris sẽ:

1. hook import system
2. lấy bytecode
3. inject tracking callbacks
4. gửi coverage về LibFuzzer

---

### Ví dụ inject

Code gốc:

```python
if x == 1337:
    danger()
```

Sau instrumentation:

```python
atheris_track(1)

if x == 1337:
    atheris_track(2)
    danger()
```

---

## 4.3. Unified Coverage

Điểm mạnh nhất của Atheris:

```text
Python coverage + native coverage
```

được merge lại.

---

### Ví dụ

```

```

=> fuzzing Python vẫn có thể trigger:

- heap overflow

- native crash

## 5.2. Minimal Harness

```python
import atheris
import sys

def test_target(data):
    text = data.decode("utf-8", errors="ignore")

    if text.startswith("M") and text.endswith("4"):
        raise AssertionError("Boom")

def TestOneInput(data):
    test_target(data)

atheris.Setup(sys.argv, TestOneInput)
atheris.Fuzz()
```

## 5.5. Deep Dive từ log thực tế

### ⚠️ Instrumentation Warning

```text
WARNING: no interesting inputs were found so far.
```

Nguyên nhân:

- chưa gọi `atheris.instrument_all()`
- LibFuzzer đang fuzz “mù”
- không có coverage feedback

---

### 🧬 Mutation Strategy

Ví dụ log:

```text
MS: 5 ChangeByte-InsertByte-InsertRepeatedBytes
```

Tức là LibFuzzer đang:

- sửa byte
- thêm byte
- lặp byte
- mutate liên tục

---

### 💥 Payload gây crash

```text
MMMMMMMMMMMM@4
```

Payload này vô tình match:

```python
startswith("M")
endswith("4")
```

---

```bash
python my_fuzzer.py crash-xxxx
```

Lúc này:

- không fuzz nữa
- chỉ replay đúng payload đã crash
- giúp reproduce bug 100%
